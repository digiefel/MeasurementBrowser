# Figure Scripts

Figure scripts are generated by the MeasurementBrowser GUI.
They are a reproducible starting point for publication figures, not the final figure itself.

Run one with:

```bash
julia --project=/path/to/MeasurementBrowser /path/to/generated_script.jl
```

## GUI Workflow

1. Select measurements in the existing MeasurementBrowser panels.
2. Open `Figure Script` from the menu bar.
3. Choose an output directory.
4. Enter a script name.
5. Create groups from the current selection.
6. Generate the script.

## What The Script Contains

The generated script defines these top-level values:

- `root_path::String`
  The measurement root used to resolve metadata and scan the stored source files.
- `project`
  The project parser/loader, for example `MeasurementBrowser.RUO2_PROJECT`.
- `source_files::Vector{String}`
  The union of source files needed by the stored groups.
- `groups::Vector{MeasurementBrowser.NamedMeasurementGroup}`
  The fixed set of named groups captured from the GUI.
- `data::MeasurementBrowser.FigureScriptData`
  The loaded and analyzed logical measurements for those groups.

`groups` is the fixed set of named groups that the GUI stored when the script was generated.

## What A Group Is

`NamedMeasurementGroup` is a small typed container:

- `name::String`
- `filter::MeasurementBrowser.MeasurementGroupFilter`

The name is the lookup key in `data`:

```julia
data["control"]
data["fatigued"]
```

Only groups that were written into the script exist. `data["new_name"]` does not invent a new filter.

`MeasurementGroupFilter` is an OR of `MeasurementFilterClause` values.
Each clause can match on:

- exact source file
- measurement kind
- exact device path or device-path prefix
- exact parameter equalities

## What `data` Is

`data` is not a table and not a dataframe.

`FigureScriptData` stores the loaded logical measurements and the fixed group matches.

Each `FigureMeasurement` has:

- `measurement::MeasurementInfo`
- `parameters::Dict{Symbol,Any}`
- `loaded`
- `analyzed`

Use it like this:

```julia
include("my_figure_script.jl")

reference = data["reference"]
first_entry = first(reference)

first_entry.measurement
first_entry.parameters
first_entry.loaded
first_entry.analyzed
```

`parameters` is the merged parameter dictionary for one logical measurement:

- `measurement.device_info.parameters`
- merged with `measurement.parameters`

## Example

```julia
using CairoMakie
include("my_figure_script.jl")

control = data["control"]

fig = Figure()
ax = Axis(fig[1, 1])

ys = [entry.parameters[:temperature_K] for entry in control if haskey(entry.parameters, :temperature_K)]

scatter!(ax, 1:length(ys), ys)
save("figure.pdf", fig)
```

Replace the field names with the ones that matter for your measurement type.
The generated script gives you the fixed groups and the loaded/analyzed measurement data.
